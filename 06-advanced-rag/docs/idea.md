# Идея проекта

**Ника** — дружелюбная Telegram-ассистентка для человека с диабетом.
Помогает вести учёт питания и инсулина, оценивать ХЕ, рассчитывать доколку,
анализировать динамику сахара и отвечать на **справочные вопросы** по руководству
для детей с сахарным диабетом 1 типа (RAG).

## Для кого

Один пользователь с сахарным диабетом, которому нужен спокойный помощник
в повседневных вопросах: что съел, сколько ХЕ, какая доколка, как идут сахара,
а также справочная информация из медицинского руководства.

## Что делает Ника

### Учёт

- Ведёт таблицу приёмов пищи, сахаров и инсулина
- Заполняет её из сообщений в чате — текст, фото, голос
- Хранит записи в JSON-файле на диске

### Расчёты

- Оценивает БЖУ и ХЕ по описанию или фото блюда
- Рекомендует дозу инсулина на еду с учётом времени доколки
- Использует коэффициенты пользователя или стандартные значения из конфига

### Отчёты

- По запросу: динамика сахаров, ХЕ и инсулина за день/неделю
- Анализ эффективности доколок

### Справочник (Advanced RAG)

- Отвечает на **справочные вопросы** по содержанию PDF-руководства и JSON-справочников
- Источник: `data/rukovodstvo_dlya_detei_i_ih_roditelei_saharnii_diabet_1_tipa_chto_neobhodimo_znat_178.pdf` + JSON в `data/`
- Перед поиском переформулирует запрос с учётом истории диалога (query transformation)
- **Три режима retrieval** — переключаются через конфиг:
  - `semantic` — семантический поиск (как раньше)
  - `hybrid` — семантика + BM25 с объединением результатов
  - `hybrid_rerank` — гибрид + cross-encoder reranker для точного ранжирования
- Ответ строится на найденных фрагментах руководства + контексте беседы
- Команда `/index` — полная переиндексация; `/index_status` — число чанков
- При старте бота — автоматическая переиндексация

**Мониторинг и качество:** при `SHOW_SOURCES=true` бот показывает документы и страницы,
использованные в ответе; все RAG-запросы трейсятся в LangSmith через переменные `LANGSMITH_*`.
Для оценки качества — синтез Q&A-датасетов из документов (`make dataset`) и RAGAS-метрики
с загрузкой feedback в LangSmith (команда `/evaluate_dataset`).

**Маршрутизация:** сообщение с данными о еде → учёт; справочный вопрос → RAG.
Подробнее — `vision.md` §6, §11, §14.

### Каналы ввода

| Канал | Что делает |
|-------|------------|
| Текст | «Съел овсянку 200 г, сахар 6.2, доколю 3 ЕД за 15 мин» → учёт; «Что такое гипогликемия?» → RAG |
| Фото | Визуальная оценка порции и состава (продукты и блюда, не упаковки) |
| Голос | Whisper API → текст на русском → тот же поток, что для текста |

### Голосовые сообщения

Пользователь надиктовывает приём пищи или вопрос — Ника распознаёт речь и отвечает
как на текстовое сообщение (учёт или RAG — по смыслу).

**Подход:** OpenAI Whisper API через OpenRouter (`openai/whisper-1`).

**Почему Whisper API:**
- Telegram Bot API **не даёт** ботовой транскрибации (только ручная кнопка у Premium)
- Простая интеграция: 1 HTTP-запрос, без ffmpeg/pydub
- Хороший русский (WER 7–11%), дёшево (~0.5 ₽ за голосовое)
- Telegram `.ogg` Opus принимается напрямую

**Поток:** голосовое → скачать файл → Whisper (`language=ru`) → текст → учёт / RAG.

**Trade-off:** аудио уходит в облако OpenRouter. Для полной приватности — локальный
`faster-whisper` (вне текущего scope).

## Как общается

- Представляется как «Ника», когда уместно (первое сообщение или спросили имя)
- От первого лица, женский род: «рада», «поняла», «подскажу», «готова помочь»
- Спокойно, поддерживающе
- Уточняет, если данных мало
- В RAG-ответах опирается на руководство; если в базе нет ответа — честно говорит об этом

## Ограничения

- Рекомендации, не медицинские назначения — решение всегда за пользователем
- Дисклеймер: «Информация справочная и не заменяет консультацию специалиста»
- Медданные можно держать локально (Ollama) — без отправки во внешние LLM
- RAG-ответы — только по проиндексированным документам, без «додумывания» фактов
