# Техническое видение проекта

> LLM-ассистент в виде Telegram-бота. Ведёт диалог с пользователем, отвечает на вопросы с ролью, заданной в системном промпте.

---

## 1. Технологии

### Язык и рантайм

- **Python 3.12**

### Управление зависимостями

- **uv** — единственный инструмент для управления проектом:
  - `pyproject.toml` — декларация зависимостей и метаданных
  - `uv.lock` — фиксация версий
  - создание и активация venv
  - установка пакетов (`uv sync`, `uv add`)

### Основные зависимости

| Пакет | Назначение |
|-------|------------|
| `aiogram` 3.x | Telegram Bot API, метод polling |
| `openai` | Клиент для LLM через OpenRouter (совместимый API) |
| `python-dotenv` | Загрузка переменных окружения из `.env` |

### Инструменты качества кода

| Инструмент | Назначение |
|------------|------------|
| `ruff` | Линтинг и форматирование |
| `mypy` | Статическая проверка типов |

### LLM

- Провайдер: **OpenRouter**
- Модель: **`openai/gpt-4o-mini`** (дёшево, достаточно для MVP)
- Клиент: `openai` SDK с кастомным `base_url`

### Сборка и запуск

| Инструмент | Назначение |
|------------|------------|
| `Makefile` | Единая точка входа: установка, запуск, Docker |
| `Docker` | Локальный запуск в контейнере (один `Dockerfile`) |

### Что намеренно не используем (MVP)

- FastAPI / веб-фреймворки
- База данных, Redis, очереди
- ORM
- CI/CD

---

## 2. Принципы разработки

### KISS

- Минимум абстракций — только то, что нужно для работы бота
- Без паттернов «на будущее» (Repository, Factory, DI-контейнеры)
- Код читается сверху вниз без прыжков по десятку файлов

### ООП: 1 класс = 1 файл

- Каждый класс — отдельный файл с тем же именем: `llm_client.py` → `class LlmClient`
- Исключение: `dataclass` / `TypedDict` для простых структур — в том же файле, где используются

### Слои (минимум)

| Слой | Назначение |
|------|------------|
| `handlers/` | Реакция на события Telegram |
| `services/` | Бизнес-логика (LLM, история диалога) |
| `config.py` | Настройки из env |
| `main.py` | Точка входа, запуск polling |

### Async

- Весь код асинхронный: `aiogram` + `openai` async client

### История диалога

- In-memory: `dict[user_id → list[message]]`
- Без персистентного хранилища на этапе MVP
- История сбрасывается при перезапуске бота

### Типизация

- Аннотации типов везде
- `mypy` в strict-режиме (или близко к нему)

### Именование

- `snake_case` — файлы, функции, переменные
- `PascalCase` — классы

### Тесты

- На старте без тестов; добавим при необходимости
