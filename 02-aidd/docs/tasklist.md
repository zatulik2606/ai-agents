## Итерации

### 0. Скелет проекта

- [ ] Структура каталогов и точка входа
- [ ] Зависимости через uv
- [ ] Бот запускается без ошибок

**Проверка:** `make run` (или аналог) — процесс стартует, polling активен.

---

### 1. Telegram: бот онлайн

- [ ] Подключение к Telegram Bot API (polling)
- [ ] Ответ на `/start`
- [ ] Ответ на текстовое сообщение (заглушка)

**Проверка:** написать боту в Telegram — получить ответ-заглушку.

---

### 2. Конфигурация

- [ ] Настройки из переменных окружения
- [ ] Шаблон `.env.example`

**Проверка:** бот стартует с токеном из `.env`, без хардкода секретов.

---

### 3. LLM-ответы

- [ ] Запрос к OpenRouter через openai-клиент
- [ ] Ответ модели отправляется пользователю

**Проверка:** задать вопрос боту — получить осмысленный ответ от LLM.

---

### 4. Системный промпт

- [ ] Роль ассистента задаётся в system prompt
- [ ] Бот отвечает в заданной роли

**Проверка:** убедиться, что тон и поведение соответствуют роли, а не дефолтной модели.

---

### 5. История диалога

- [ ] Контекст сохраняется в памяти per user
- [ ] Бот учитывает предыдущие сообщения

**Проверка:** задать уточняющий вопрос — бот отвечает с учётом контекста.

---

### 6. Команды бота

- [ ] `/start` — приветствие
- [ ] `/reset` — сброс истории
- [ ] `/help` — краткая справка

**Проверка:** `/reset` очищает контекст; следующий ответ не ссылается на прошлый диалог.

---

### 7. Make + Docker

- [ ] Makefile: install, run, docker-build, docker-run
- [ ] Dockerfile для локального запуска

**Проверка:** бот работает через `make run` и через `make docker-run`.

---

### 8. Логирование

- [ ] Логи старта, входящих сообщений, ответов LLM
- [ ] Логирование ошибок без падения бота

**Проверка:** при ошибке LLM пользователь видит понятное сообщение; детали — в логах.
