# Идея проекта

**Ника** — дружелюбная Telegram-ассистентка для человека с диабетом.
Помогает вести учёт питания и инсулина, оценивать ХЕ, рассчитывать доколку,
анализировать динамику сахара и отвечать на **справочные вопросы** по руководству
для детей с сахарным диабетом 1 типа через **автономного ReAct-агента** с инструментом поиска.

## Для кого

Один пользователь с сахарным диабетом, которому нужен спокойный помощник
в повседневных вопросах: что съел, сколько ХЕ, какая доколка, как идут сахара,
а также справочная информация из медицинского руководства.

## Что делает Ника

### Учёт

- Ведёт таблицу приёмов пищи, сахаров и инсулина
- Заполняет её из сообщений в чате — текст, фото, голос
- Хранит записи в JSON-файле на диске

### Расчёты

- Оценивает БЖУ и ХЕ по описанию или фото блюда
- Рекомендует дозу инсулина на еду с учётом времени доколки
- Использует коэффициенты пользователя или стандартные значения из конфига

### Отчёты

- По запросу: динамика сахаров, ХЕ и инсулина за день/неделю
- Анализ эффективности доколок

### Справочник (ReAct-агент + Advanced Retrieval + MCP)

- Отвечает на **справочные вопросы** по PDF-руководству и JSON-справочникам
- Источник: `data/rukovodstvo_dlya_detei_i_ih_roditelei_saharnii_diabet_1_tipa_chto_neobhodimo_znat_178.pdf` + JSON в `data/`
- Точка входа для справочного диалога — **LangChain-агент** (`create_agent` + `MemorySaver`)
- Агент сам решает, какой tool вызвать, и **сам формулирует** аргументы
  (отдельный узел query transformation не нужен)
- Локальные tools:
  - **`rag_search`** — поиск по индексу (`semantic` | `hybrid` | `hybrid_rerank`), ответ генерирует агент
  - **`glucose_unit_converter`** — ммоль/л ↔ мг/дл (фиксированный коэффициент)
- Tools через **MCP** (сервер `mcp-nika-agent`, см. `vision.md` §16):
  - **`search_products`** — средства и расходники для диабета (глюкометры, полоски,
    инсулиновые ручки/картриджи, CGM, помпы); данные динамические, их нет в PDF/JSON
    головного `data/`
  - **`food_nutrition`** — онлайн-поиск КБЖУ/углеводов по продукту питания
    (Open Food Facts API)
  - **`calculate_meal_bolus`** — оценка ХЕ и болюса по углеводам
    (опционально коррекция по сахару и БЖЕ); справочный расчёт, не назначение
- Если MCP-сервер недоступен — агент работает с локальными tools (graceful degradation)
- Команда `/index` — полная переиндексация; `/index_status` — число чанков
- При старте бота — автоматическая переиндексация

**Мониторинг и качество:** при `SHOW_SOURCES=true` бот показывает документы и страницы,
использованные в ответе; RAG/agent-запросы трейсятся в LangSmith через `LANGSMITH_*`.
Для оценки качества — синтез Q&A-датасетов (`make dataset`) и RAGAS-метрики
с feedback в LangSmith (`/evaluate_dataset`). Evaluation гоняет тот же агент
и сохраняет перечень документов из tool-вызовов текущего ответа.

**Маршрутизация:** сообщение с данными о еде → учёт; справочный / свободный вопрос → агент.
Подробнее — `vision.md` §6, §11, §14–16.

**Запуск со всеми tools** (два терминала, порядок важен):

```bash
make run-mcp-nika   # Terminal 1: MCP-сервер :8000
make run            # Terminal 2: Telegram-бот
```

### Каналы ввода

| Канал | Что делает |
|-------|------------|
| Текст | «Съел овсянку…» → учёт; «Что такое гипогликемия?» → `rag_search`; «Какие полоски к Contour?» → `search_products`; «Сколько углеводов в греческом йогурте?» → `food_nutrition` |
| Фото | Визуальная оценка порции и состава (продукты и блюда, не упаковки) |
| Голос | Whisper API → текст на русском → тот же поток, что для текста |

### Голосовые сообщения

Пользователь надиктовывает приём пищи или вопрос — Ника распознаёт речь и отвечает
как на текстовое сообщение (учёт или агент — по смыслу).

**Подход:** OpenAI Whisper API через OpenRouter (`openai/whisper-1`).

**Почему Whisper API:**
- Telegram Bot API **не даёт** ботовой транскрибации (только ручная кнопка у Premium)
- Простая интеграция: 1 HTTP-запрос, без ffmpeg/pydub
- Хороший русский (WER 7–11%), дёшево (~0.5 ₽ за голосовое)
- Telegram `.ogg` Opus принимается напрямую

**Поток:** голосовое → скачать файл → Whisper (`language=ru`) → текст → учёт / агент.

**Trade-off:** аудио уходит в облако OpenRouter. Для полной приватности — локальный
`faster-whisper` (вне текущего scope).

## Как общается

- Представляется как «Ника», когда уместно (первое сообщение или спросили имя)
- От первого лица, женский род: «рада», «поняла», «подскажу», «готова помочь»
- Спокойно, поддерживающе
- Уточняет, если данных мало
- В ответах по руководству опирается на `rag_search`; по расходникам —
  на `search_products`; по КБЖУ еды — на `food_nutrition`
- Если нужных данных в tool нет — честно говорит об этом

## Ограничения

- Рекомендации, не медицинские назначения — решение всегда за пользователем
- Дисклеймер: «Информация справочная и не заменяет консультацию специалиста»
- Медданные можно держать локально (Ollama) — без отправки во внешние LLM
- Факты по руководству — только из `rag_search`; расходники — из MCP
  `search_products`; питание — из `food_nutrition`, без «додумывания» вне результатов tools
- MCP-сервер — отдельный процесс; без него бот жив, но без MCP-tools
