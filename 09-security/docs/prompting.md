# Концепция промптинга для Ники

> Как выбирать стратегию few-shot / zero-shot для разных задач бота.

---

## Парадигмы

| Парадигма | Что в промпте | Когда использовать |
|-----------|---------------|-------------------|
| **Zero-shot** | Только инструкция, без примеров | Простая роль, свободный диалог, задачи где модель уже «знает» домен |
| **One-shot** | 1 пример «вход → выход» | Нужен конкретный формат ответа или однотипная логика |
| **Few-shot** | 2–5 примеров | Structured output, пограничные случаи, неоднозначные формулировки |
| **Many-shot** | 10+ примеров | Редко: сложная классификация, нестандартный жаргон; дорого по токенам |

**Принцип KISS для Ники:** начинать с zero-shot → добавлять примеры только если модель стабильно ошибается.

---

## Карта промптов Ники

| Промпт | Файл | Сейчас | Почему |
|--------|------|--------|--------|
| Диалог (`DEFAULT_SYSTEM_PROMPT`) | `config.py` | **Zero-shot** | Роль и правила; ответы вариативны, примеры ограничат тон |
| Извлечение еды (`EXTRACTION_SYSTEM_PROMPT`) | `llm_client.py` | **Zero-shot + справочные значения** | Правила + 3 эталона БЖУ (огурец, помидор, банан) — не полноценные few-shot примеры |
| Описание фото (`PHOTO_DESCRIBE_PROMPT`) | `llm_client.py` | **One-shot (шаблон)** | Задан формат вывода, но нет готового примера «фото → ответ» |
| Парсинг фото (`PHOTO_EXTRACTION_PROMPT`) | `llm_client.py` | **Zero-shot** | Pydantic-схема задаёт структуру; примеры дублировали бы schema |
| Анализ недели (`ask_brief`) | `llm_client.py` | **Zero-shot** | Краткий комментарий к готовой сводке |
| Ответ пользователю при записи еды | `meal_log.py` | **Zero-shot (шаблоны кода)** | Формат фиксируется в Python, не в промпте |

## Рекомендации по задачам

### 1. Свободный диалог — Zero-shot

```
Роль + правила + дисклеймер
```

Не добавлять примеры диалогов: модель начнёт копировать заготовки вместо живого ответа.

### 2. Structured extraction (текст / голос) — Few-shot (2–3)

Когда zero-shot ошибается (смешение языков, неверные ХЕ, пропуск сахара):

```
Правила
---
Пример 1:
Вход: «Съел овсянку 200 г, сахар 6.2, завтрак»
Выход: { should_log: true, product: "овсянка", carbs_g: 42, ... }

Пример 2:
Вход: «Ты кто?»
Выход: { should_log: false, ... }

Пример 3:
Вход: «Что-то съел»
Выход: { should_log: true, needs_clarification: true, ... }
```

**Не больше 5 примеров** — structured output и так ограничен схемой `MealExtraction`.

### 3. Vision (описание фото) — One-shot

Один полный пример улучшает стабильность формата:

```
Пример:
Продукт: помидор
Порция: 150 г
Детали: один средний помидор на тарелке.
```

### 4. Парсинг описания фото — Few-shot (2)

Один пример «описание → JSON-поля», один — с подписью и сахаром.

### 5. Отчёты — Zero-shot

Сводка формируется кодом (`MealReport`); LLM только комментирует готовые цифры. Примеры не нужны.

## Антипаттерны

| Не делать | Почему |
|-----------|--------|
| Many-shot в каждом промпте | Раздувает контекст, дорого, модель «залипает» на примерах |
| Примеры с устаревшими коэффициентами | ХЕ и дозы считает `InsulinCalculator`, не LLM |
| Примеры ответов пользователю в промпте | Ответы — шаблоны в `MealExtraction.build_reply()` |
| Дублировать Pydantic-схему примерами | Schema + 2–3 edge-case примера достаточно |

---

## Алгоритм выбора

```
Задача новая?
  └─ Модель справляется с zero-shot?
       ├─ Да  → оставить zero-shot
       └─ Нет → ошибка в формате?
            ├─ Да  → one-shot (1 пример формата)
            └─ Нет → ошибка в логике / классификации?
                 ├─ Да  → few-shot (2–5 примеров)
                 └─ Нет → проверить схему, код, модель; many-shot — последний resort
```

---

## Связь с архитектурой

```
Промпт (zero/one/few-shot)
        ↓
   LLM → MealExtraction (Pydantic)
        ↓
   Python-шаблоны + InsulinCalculator  ← детерминированная логика, не промпт
        ↓
   Ответ пользователю
```

**Промпт отвечает за понимание входа.**  
**Код отвечает за расчёты, формат ответа и дисклеймер.**

## Чеклист перед добавлением примера

- [ ] Есть повторяющаяся ошибка на zero-shot (≥3 случая)?
- [ ] Пример покрывает edge-case, а не дублирует правило?
- [ ] Вход и выход в примере актуальны (ммоль/л, русский язык)?
- [ ] Не больше 5 примеров в одном промпте?
- [ ] Расчёт инсулина остаётся в `InsulinCalculator`, не в примере?
